### Editor, lint & format

* **clojure-lsp** (one server; code actions, rename, clean-ns, format, etc.). It uses **clj-kondo** under the hood—drop a `.clj-kondo/config.edn` in the repo and you’re off. If you’re in VS Code (Calva) or Emacs (lsp-mode), this is the baseline. ([clojure-lsp.io][1])
* **clj-kondo** CLI for CI and pre-commit (fast static analysis for CLJ/CLJS/EDN; also available via npm). ([GitHub][2])
* **Formatter**: start with **cljfmt** (simple, widely supported). If you want more knobs later, consider **zprint**. ([GitHub][3])

### REPL & data inspection (massive quality-of-life)

* **Portal** (tap>-driven inspector; works with CLJS; easy preload). Add Portal or the tiny **playback** preload to auto-open a Portal window and pipe `tap>` values there. ([practical.li][4])
* Use Shadow’s **`:preloads`** to wire dev-only helpers without touching app code. ([cljs.github.io][5])

### App & state debugging (re-frame projects)

* **re-frame-10x** (event DB, subscriptions, traces; “programmer’s dashboard”). Add as a dev dep and preload it. ([GitHub][6])
* **re-frisk** (DB & event inspection; “in-app” or remote). Also just a dev dep + preload. ([cljdoc.org][7])
* re-frame docs recommend 10x/re-frisk together with `cljs-devtools`. You already have Binaryage DevTools; keep it preloaded in dev. ([day8.github.io][8])

### Testing you’ll actually run

* **Built-in Shadow targets**:

  * `:node-test` to run CLJS tests on Node (fast local/CI).
  * `:karma` for browser CI (ChromeHeadless, etc.).
    See the user guide and concise blog walkthroughs. ([shadow-cljs.github.io][9])
* If you prefer browser CI from day one, follow the Karma recipe (deps + `karma.conf.js`) and `:ns-regexp "-test$"`. ([abhinavomprakash.com][10])
* (Note: CLJS line-coverage tooling is still thin; people approximate with namespace-level checks or run CLJ coverage where applicable.) ([ag91.github.io][11])

### E2E/UI testing (optional but nice)

* **Playwright** or **Cypress** to drive the app end-to-end in CI; your app is just JS in the browser, so these work fine alongside Shadow. ([GitHub][12])

### Emacs/Spacemacs integration

* If you’re in Spacemacs, the **Clojure layer** gives you CIDER + refactor goodies; pair it with clojure-lsp for LSP features. ([spacemacs.org][13])

## Drop-in snippets

**1) Dev dependencies & preloads (Portal + 10x + re-frisk)**

```edn
;; shadow-cljs.edn (add dev-only libs in your deps tool of choice)
{:dependencies
 [[djblue/portal "0.61.0"]            ;; inspector
  [day8.re-frame/re-frame-10x "1.10.1"]
  [re-frisk "1.7.1"]]

 :builds
 {:app
  {:target :browser
   :output-dir "resources/public/js"
   :asset-path "/js"
   :modules {:main {:init-fn opencode.ui.core/init}}
   :devtools {:http-root "resources/public"
              :http-port 8700
              :preloads [devtools.preload
                         day8.re-frame-10x.preload
                         re-frisk.preload
                         playback.preload]}}}}
```

* `:preloads` is the sanctioned way to auto-init dev helpers. `playback.preload` wires Portal without custom code; you can swap it for your own preload if you want. ([cljs.github.io][5])

**2) Tests you can run locally & in CI**

```edn
;; Add one (or both) test builds:

:builds
{:test-node
 {:target :node-test
  :output-to "target/node-tests.js"
  :ns-regexp "-test$"}

 :test-browser
 {:target :karma
  :output-to "target/ci.js"
  :ns-regexp "-test$"}}
```

* Pair `:test-browser` with a `karma.conf.js` using `ChromeHeadless` and `frameworks: ['cljs-test']`. ([shadow-cljs.github.io][9])

**3) Editor/LSP**

* Add `.clj-kondo/config.edn` (rules, suppressions) and turn on clojure-lsp in your editor; Calva already pipes linting from clojure-lsp/clj-kondo. For Emacs, use `lsp-mode` with clojure-lsp. ([calva.io][14])

## Quick picks (if you only add 3 things)

1. **clojure-lsp + clj-kondo** (editor + CI lint/format). ([clojure-lsp.io][1])
2. **Portal** via `:preloads` (see results of `tap>` instantly). ([practical.li][4])
3. **node-test** target (fast tests you’ll actually run). ([blog.cofx.nl][15])

If you’re using re-frame, tack on **re-frame-10x** immediately. ([GitHub][6])
