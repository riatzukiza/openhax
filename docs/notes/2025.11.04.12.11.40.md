### Inspector & Debugging

* **Portal** — value inspector that works great from CLJS.
  *Why:* tap> any data and explore it in a pane; handy for event payloads and app-db slices. ([GitHub][1])
  **deps.edn**: `djblue/portal {:mvn/version "0.58.5"}`
  **shadow-cljs.edn** (dev only):

  ```clojure
  :devtools {:preloads [portal.shadow.preload]}
  ```
* **re-frame-10x** (if you’re on re-frame) — time-traveling event/view debugger. ([docs.cider.mx][2])
* **Re-frisk** — lightweight Reagent/re-frame state inspector if you don’t need the full 10x. ([GitHub][3])
* **FlowStorm** — modern step debugger with CLJS support; great for gnarly logic. ([cljdoc.org][4])
* **Shadow “Inspect UI” / Browser Extension** — ships with shadow; shows builds/modules, REPLs, logs. ([code.thheller.com][5])
* **Binaryage tools** — you already use `cljs-devtools`; consider **Dirac** if you want a Chrome-DevTools fork tuned for CLJS REPL + breakpoints (note: extension availability has changed; still useful in certain setups). ([GitHub][6])

### Linting & Format

* **clj-kondo** — fast linting for Clojure/Script & EDN; integrates with editors and CI. ([practical.li][7])
* **cljfmt** (via clojure-lsp) — opinionated formatter; configure once per repo. ([clojure-lsp.io][8])
* **zprint** — alternative pretty printer/formatter (nice for CI checks). ([ClojureVerse][9])

### Testing (unit, DOM, coverage)

* **cljs-test-runner** — discover & run CLJS tests (Node or browser). Simple to wire into CI. ([GitHub][10])
* **Kaocha** — unified test runner (CLJ/CLJS) with plugins: JUnit XML, Cloverage coverage.
  *Plugins:* `kaocha-junit-xml`, `kaocha-cloverage`. ([GitHub][11])
* **Coverage** — Cloverage (via kaocha-cloverage) for reports you can surface in CI. ([cljdoc.org][12])
* **DOM testing** — run CLJS tests against the DOM (e.g., testing-library style) or a browser via Karma; both patterns are used with shadow-cljs. ([arthurbrrs.me][13])

*Minimal example (deps.edn):*

```clojure
{:aliases
 {:test {:extra-deps {olical/cljs-test-runner {:mvn/version "3.8.1"}
                      lambdaisland/kaocha     {:mvn/version "1.91.1392"}
                      lambdaisland/kaocha-junit-xml {:mvn/version "1.17.101"}
                      lambdaisland/kaocha-cloverage {:mvn/version "1.1.89"}}
         :main-opts  ["-m" "kaocha.runner"]}}}
```

(Produces JUnit XML for CI and lets you add coverage gates.) ([Clojars][14])

### API Mocking / Offline Dev

* **MSW (Mock Service Worker)** — intercept HTTP at the network layer in the browser & Node; dev and tests share the same mocks. This pairs well with `cljs-ajax` during local dev. ([mswjs.io][17])

### Error Reporting & Perf

* **Sentry (browser SDK)** — production error & performance tracing; straightforward from CLJS interop. ([docs.sentry.io][18])
  *Example (CLJS):*

  ```clojure
  (ns app.sentry
    (:require ["@sentry/browser" :as Sentry]))
  (defn init! [dsn]
    (.init Sentry #js {:dsn dsn}))
  ```

  (Many teams wrap this in a Reagent error boundary; examples exist using re-frame too.) ([DEV Community][19])

### Shadow-CLJS niceties you can flip on

* **Build report** (you already added the hook) — keep using it to watch bundle drift over time. ([playwright.dev][24])
* **:preloads** for dev helpers (portal, re-frame-10x, re-frisk). Preloads are the canonical way to auto-mount tools during `watch`. ([GitHub][25])
* **Inspect UI / Browser Extension** for quick visibility into modules, loaded builds, logs. ([code.thheller.com][5])

---

### Example: dev-only preloads bundle

```clojure
;; shadow-cljs.edn (fragment)
{:dependencies [;; … your libs …
                [djblue/portal "0.58.5"]]
 :builds
 {:app {:target     :browser
        :modules    {:main {:init-fn opencode.ui.core/init}}
        :devtools   {:http-root "resources/public"
                     :http-port 8700
                     :preloads  [portal.shadow.preload]}}}}
```

(Replace/add `re-frame-10x.preload` or `re_frisk.preload` as needed.) ([docs.cider.mx][2])
